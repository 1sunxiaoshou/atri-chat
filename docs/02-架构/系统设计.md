# 系统设计

本文档记录当前已经落地的系统设计。未来计划见 [路线图](../03-规划/路线图.md)。

## 1. 系统定位

ATRI Chat 是 Windows 桌面优先的 AI 伴侣应用：

```text
Tauri 桌面壳
 -> React 前端
 -> FastAPI 后端
 -> LangChain / LangGraph Agent
 -> SQLite / 文件资源 / ASR / TTS / VRM 前端执行层
```

核心能力：

- 角色化 LLM 聊天
- 长期记忆
- VRM 3D 角色渲染与动作控制
- ASR 语音识别
- TTS 语音合成
- 本地状态与资源管理

## 2. 分层职责

| 层 | 职责 | 主要位置 |
|---|---|---|
| 桌面壳 | 启动后端、分配端口、注入运行时目录 | `frontend/src-tauri/` |
| 前端 | 聊天、管理页、VRM 渲染、状态展示 | `frontend/` |
| 后端 API | 健康检查、资源、配置、会话接口 | `api/`、`core/bootstrap.py` |
| Agent | 模型调用、工具、提示词、消息持久化 | `core/agent/`、`core/middleware/` |
| 状态与资源 | SQLite、记忆文件、静态资源、运行状态 | `core/db/`、`data/` |

当前主链：

```text
启动
 -> /health
 -> /agent-stream
 -> useAgentStream
 -> 文本聊天 / VRM 命令执行
 -> runtime status / feedback
```

## 3. 启动与运行时目录

Tauri 是桌面模式的运行时契约来源，向 Python 注入：

- `ATRI_BACKEND_PORT`
- `ATRI_RUNTIME_MODE`
- `ATRI_APP_ROOT`
- `ATRI_DATA_ROOT`

Python 侧规则：

- `app_env` 从 `ATRI_RUNTIME_MODE` 派生。
- 日志目录固定为 `data_root/logs`。
- Python 创建日志、数据库、静态资产、模型、记忆和模态资源目录。

关键文件：

- `main.py`
- `core/bootstrap.py`
- `core/runtime_layout.py`
- `core/config.py`
- `core/route_registry.py`
- `frontend/src-tauri/src/runtime_layout.rs`
- `frontend/src-tauri/src/backend.rs`

## 4. 数据与资源

重要目录：

| 目录 | 用途 |
|---|---|
| `data/sqlite/` | 应用数据库、LangGraph checkpoints、长期 store |
| `data/logs/` | 日志 |
| `data/memory/` | 角色长期记忆 |
| `data/models/asr/` | ASR 模型 |
| `data/assets/vrm/models/` | VRM 模型 |
| `data/assets/vrm/motions/` | VRM 动作 |

主要表：

- `provider_configs`
- `models`
- `characters`
- `conversations`
- `messages`
- `assets_avatars`
- `assets_motions`
- `character_motion_bindings`
- `tts_providers`
- `assets_voices_v2`

尚未作为当前事实的对象：会话提示词快照、工具集快照、持久化 Agent 运行事件、资源 manifest / materialization。

## 5. 聊天主链

当前聊天入口统一为：

```text
frontend/hooks/useAgentStream.ts
 -> POST /api/v1/agent-stream
 -> core/agent/coordinator.py
 -> LangChain / LangGraph Agent
 -> SSE event
 -> @langchain/react useStream
```

约定：

```text
conversation_id == thread_id
```

这使业务会话、LangGraph checkpoint 和前端 thread 使用同一标识。

## 6. Agent、提示词与工具

Agent 由 `core/agent/factory.py` 创建，主要中间件：

| 中间件 | 职责 |
|---|---|
| `dynamic_model.py` | 每次请求选择真实模型 |
| `dynamic_prompt.py` | 根据角色和模式构建提示词 |
| `dynamic_tools.py` | 按模式过滤工具 |
| `persist_messages.py` | 持久化消息 |

提示词模板：

- `core/prompts/templates/identity.md`
- `core/prompts/templates/normal.md`
- `core/prompts/templates/vrm.md`

当前提示词仍是运行时构建，尚未实现会话级 system prompt snapshot。

工具规则：

- 所有模式允许 `memory_*`。
- VRM 模式额外允许 `perform_actions` 和 `control_camera`。

命令系统见 [命令系统](./命令系统.md)。

## 7. VRM 执行链

VRM 控制由后端发送意图、前端执行：

```text
AI 调用工具
 -> 后端写 custom event
 -> useAgentStream 接收事件
 -> ChatInterface 转交 VRM 逻辑
 -> useVRM 解析并执行
 -> runtime feedback 回传后端
```

主要前端入口：

- `frontend/components/chat/ChatInterface.tsx`
- `frontend/components/vrm/logic/useVRM.ts`
- `frontend/store/vrm/useVRMStore.ts`

## 8. 运行时状态

运行时状态由 `CapabilityRegistry` 聚合，当前能力：

- `agent`
- `asr`
- `tts`
- `vrm`

接口：

- `GET /api/v1/runtime/status`
- `GET /api/v1/runtime/events`
- `POST /api/v1/runtime/vrm-feedback`

当前事件保存在内存中，进程重启后丢失。

## 9. 模态能力

ASR：

- 入口：`api/routes/asr.py`、`api/routes/asr_mgmt.py`
- 引擎：`core/asr/`
- 加载方式：惰性获取

TTS：

- 入口：`api/routes/tts.py`
- 管理：`api/routes/tts_providers.py`、`api/routes/voice_assets.py`
- 实现：`core/tts/`
- 结构：供应商配置 + 音色资产 + 角色绑定

## 10. 当前边界

已落地：

- 关键路由与延迟路由拆分
- `agent_stream + useAgentStream` 主链
- `conversation_id == thread_id`
- VRM 最小命令链
- 前端 VRM 执行反馈
- 最小 `CapabilityRegistry`
- PromptService 模板化提示词
- ASR/TTS 惰性实例和状态聚合

未落地：

- 会话级提示词快照
- 长会话搜索、FTS 和压缩
- 资源 manifest / materialization
- 持久化 Agent 运行事件
- 完整能力适配器目录结构
