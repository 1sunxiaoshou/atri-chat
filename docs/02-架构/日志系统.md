# 日志系统

本文档说明当前日志文件、配置和排查入口。

## 目录

日志目录由运行时数据目录派生：

```text
data/logs/
```

桌面模式下，Tauri 只注入 `ATRI_DATA_ROOT`，Python 固定使用 `data_root/logs`。

## 文件

| 文件 | 用途 |
|---|---|
| `app.log` | 应用主日志 |
| `error.log` | 错误日志 |
| `access.log` | HTTP 访问日志 |
| `llm_calls.log` | LLM 调用日志，按配置启用 |
| `slow_queries.log` | 慢请求或慢查询记录 |

## 配置

常用环境变量：

| 变量 | 用途 |
|---|---|
| `LOG_LEVEL` | 日志级别，默认通常为 `INFO` |
| `ENABLE_HTTP_LOGGING` | 是否记录 HTTP 请求日志 |
| `ENABLE_LLM_CALL_LOGGER` | 是否记录 LLM 调用详情 |
| `SLOW_QUERY_THRESHOLD_MS` | 慢请求阈值 |

## 排查命令

PowerShell 示例：

```powershell
Get-Content data/logs/error.log -Tail 100
Get-Content data/logs/app.log -Tail 100 -Wait
Select-String -Path data/logs/*.log -Pattern "ERROR"
Select-String -Path data/logs/access.log -Pattern "/api/v1/agent-stream"
```

## 记录规则

应该记录：

- 启动阶段关键路径和异常。
- API 请求失败原因。
- Agent、ASR、TTS、VRM 关键状态变化。
- 外部服务调用失败。

不应该记录：

- API Key、Token、Cookie。
- 用户完整隐私内容。
- 大段音频、图片或模型文件内容。
- 可从上下文稳定推导出的重复噪声。
