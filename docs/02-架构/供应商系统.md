# 供应商系统

本文档说明 LLM 供应商配置、模型发现和前端展示的当前实现。

## 当前职责

供应商系统负责：

- 保存 LLM 供应商连接配置。
- 根据供应商类型创建 LangChain 模型。
- 同步或手动维护模型列表。
- 标记模型能力，如视觉、工具调用、reasoning。
- 在前端显示对应供应商图标。

## 主要代码

| 路径 | 职责 |
|---|---|
| `core/models/providers.yaml` | 内置供应商模板 |
| `core/models/providers/` | 供应商适配代码 |
| `core/repositories/provider_repository.py` | 供应商持久化访问 |
| `api/routes/providers.py` | 供应商 API |
| `core/middleware/dynamic_model.py` | 每次请求选择真实模型 |
| `frontend/services/api/providers.ts` | 前端供应商 API |

## 数据对象

| 对象 | 用途 |
|---|---|
| `provider_configs` | 供应商实例配置 |
| `models` | 模型配置与能力标记 |

`provider_configs` 保存用户配置，例如 API Key、Base URL、供应商类型。`models` 保存可选模型及其能力，用于聊天时选择模型和构建运行时参数。

## API

| 路径 | 用途 |
|---|---|
| `GET /api/v1/providers` | 列出供应商配置 |
| `POST /api/v1/providers` | 创建供应商配置 |
| `GET /api/v1/providers/{id}` | 查看供应商配置 |
| `PUT /api/v1/providers/{id}` | 更新供应商配置 |
| `DELETE /api/v1/providers/{id}` | 删除供应商配置 |
| `GET /api/v1/providers/templates/list` | 获取供应商模板 |
| `GET /api/v1/providers/{id}/models` | 获取已保存模型 |
| `POST /api/v1/providers/{id}/sync` | 同步模型 |
| `GET /api/v1/providers/{id}/available-models` | 查询远端可用模型 |

## 模型选择链路

```text
前端选择 provider_config_id + model_id
 -> /agent-stream context
 -> dynamic_model middleware
 -> 根据供应商配置创建 LangChain ChatModel
 -> Agent 本轮请求使用该模型
```

## 图标

前端使用 `@lobehub/icons` 根据 `provider_type` 显示图标。后端不再保存 logo 字段。

## 新增供应商

新增供应商时检查：

1. `core/models/providers.yaml` 是否需要新增模板。
2. `core/models/providers/` 是否需要新增适配器。
3. 模型发现方式是 OpenAI-compatible、Ollama、Google，还是不支持发现。
4. `dynamic_model` 是否能创建对应 LangChain 模型。
5. 前端是否需要新增特殊配置字段。

## 维护规则

- API Key 等敏感字段只放在供应商配置中。
- 模型能力标记属于 `models`，不要写死在前端。
- 能使用 OpenAI-compatible 适配的供应商优先复用通用逻辑。
- 前端图标以 `provider_type` 为准，不恢复后端 logo 字段。
