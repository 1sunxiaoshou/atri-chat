# 命令系统

本文档说明当前 VRM 命令系统。命令系统是已落地架构，不再放在规划目录。

## 1. 定位

命令系统让 AI 在 VRM 模式下驱动角色表现。后端只发送意图，前端负责解析、执行和反馈。

原则：

- 命令保持轻量。
- AI 不直接操作渲染对象。
- 普通文本模式不暴露 VRM 命令工具。
- 不恢复旧的文本内嵌标签协议。

## 2. 后端工具

工具定义：`core/tools/action_tools.py`

| 工具 | 用途 |
|---|---|
| `perform_actions(commands: string[])` | 发送一组顺序执行的角色表现命令 |
| `control_camera(action, preset, durationMs?)` | 发送镜头控制意图 |

工具通过 `ToolRuntime.stream_writer` 写出 custom event：

- `vrm.perform_actions`
- `vrm.control_camera`

## 3. 暴露规则

工具过滤：`core/middleware/dynamic_tools.py`

| 模式 | 可用工具 |
|---|---|
| 文本模式 | `memory_*` |
| VRM 模式 | `memory_*`、`perform_actions`、`control_camera` |

预留 `vrm_*` 作为未来 VRM 专用工具前缀。

## 4. 命令语法

`perform_actions` 接收字符串数组。

| 命令 | 格式 | 语义 |
|---|---|---|
| 表情 | `emotion <name>` | 切换表情 |
| 动作 | `motion <name>` | 播放动作 |
| 说话 | `say <emotion> | <text>` | 设置表情和字幕，播放或等待语音段 |
| 等待 | `wait <ms>` | 停顿 |

示例：

```text
[
  "emotion happy",
  "say happy | 欢迎回来",
  "wait 300",
  "motion wave"
]
```

无效命令会被丢弃。整批命令都无效时，工具返回 `error: invalid command`。

## 5. TTS 关系

`perform_actions` 会为 `say` 命令尝试生成角色 TTS 音频：

```text
say command
 -> synthesize_character_speech_file()
 -> custom event speech[].audioUrl
 -> useVRM 播放音频
```

合成失败时不阻断命令，前端按文本长度等待。

## 6. 前端执行链

主要文件：

- `frontend/hooks/useAgentStream.ts`
- `frontend/components/chat/ChatInterface.tsx`
- `frontend/utils/actionCommands.ts`
- `frontend/components/vrm/logic/useVRM.ts`

流程：

```text
custom event
 -> useAgentStream
 -> ChatInterface
 -> actionCommands parser
 -> useVRM
 -> runtime feedback
```

执行语义：

- `emotion`：立即设置表情。
- `motion`：在角色动作绑定中查找并播放动作。
- `say`：设置表情和字幕，优先播放音频。
- `wait`：等待指定毫秒数。
- `control_camera`：写入相机运行状态，不进入角色表现队列。

## 7. 提示词约束

VRM 模式提示词：`core/prompts/templates/vrm.md`

当前约束：

- 正文保持自然语言。
- 动作、表情、镜头通过工具完成。
- 只使用当前角色真实存在的表情和动作 ID。

## 8. 检查

解析器检查：

```bash
npm --prefix frontend run check-commands
```

修改命令语法时同时检查：

- 后端 `_parse_action_command`
- 前端 `parseActionCommand`
- VRM 模式提示词
- `check-action-commands.ts`
