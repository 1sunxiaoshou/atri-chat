# 记忆系统架构

本文档说明当前长期记忆系统。短期对话状态由 LangGraph checkpointer 管理，不在本文展开。

## 当前模型

长期记忆按角色隔离，保存在：

```text
data/memory/{character_id}/
```

记忆文件使用 Markdown。Agent 通过工具读写相对路径，不能越过当前角色的记忆目录。

## 主要代码

| 路径 | 职责 |
|---|---|
| `core/tools/memory_tools_v3.py` | 记忆工具 |
| `core/prompts/service.py` | 将记忆说明注入提示词 |
| `core/agent/factory.py` | 注册记忆工具 |
| `core/middleware/dynamic_tools.py` | 允许所有模式使用 `memory_*` |

## 工具

| 工具 | 用途 |
|---|---|
| `memory_read(file, section=None, tail_lines=None)` | 读取记忆文件 |
| `memory_write(file, content, section=None, mode="append")` | 写入或更新记忆 |
| `memory_search(query, file=None, limit=10)` | 搜索记忆 |
| `memory_list(file=None)` | 列出记忆文件或目录 |

## 建议文件

| 文件 | 用途 |
|---|---|
| `user_profile.md` | 用户画像、长期稳定事实 |
| `memory_stream.md` | 时间顺序事件记录 |
| `topics.md` | 主题摘要和归纳 |

这些文件是约定，不是硬编码限制。Agent 可以在角色目录内创建子目录和专题文件。

## 访问边界

允许：

```text
user_profile.md
archive/2026-04.md
topics/projects.md
```

不允许：

```text
../other_character/user_profile.md
C:/Users/name/file.md
/absolute/path/file.md
```

## 提示词关系

当前系统会在角色提示词中说明可用记忆工具。具体工具是否暴露由 `dynamic_tools` 控制：

- 文本模式：允许 `memory_*`
- VRM 模式：允许 `memory_*`

记忆不是每轮全部塞入 prompt。Agent 需要时通过工具读取、搜索或写入。

## 维护规则

- 记忆内容以角色为边界，不跨角色共享。
- 稳定事实写入 `user_profile.md`。
- 对话事件写入 `memory_stream.md` 或按日期归档。
- 主题总结写入 `topics.md` 或 `topics/*.md`。
- 不把大段原始聊天记录长期堆进画像文件。
- 修改工具参数时同步检查 prompt、测试脚本和前端调试体验。
