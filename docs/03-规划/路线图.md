# 路线图

本文档记录 ATRI Chat 后续计划。它只维护尚未完成、仍值得推进的事项；已经落地的系统设计统一放在 [系统设计](../02-架构/系统设计.md)。

## 1. 当前主线

当前优先保护和继续打磨的主链是：

```text
Tauri 启动
 -> FastAPI /health
 -> /agent-stream
 -> useAgentStream
 -> 文本聊天 / VRM 命令执行
 -> runtime status / feedback
```

已完成或基本完成的事项不再在本文展开，只在对应架构文档中作为事实记录。

## 2. 近期计划

### 2.1 文档与代码收口

- 清理过期规划文档和重复说明。
- 清除废弃代码。
- 修复 TypeScript 类型错误。
- 继续迁移仍在使用的旧 API 调用到模块化 API。

### 2.2 设置与默认资源

- 优化设置页 UI 与信息组织。
- 整理系统默认资源，使用户未配置资源时也有可用默认显示。
- 梳理默认角色、默认背景、默认动作和默认音色的兜底策略。

### 2.3 VRM 体验打磨

- 调整渲染风格，在性能可接受的前提下增强二次元、卡通观感。
- 支持 VRM 背景切换。
- 支持头像位置显示切换。
- 评估运镜系统是否需要进入正式主线。

## 3. 中期计划

### 3.1 状态管理整理

- 评估并推进更清晰的全局状态管理。
- 拆分 `App.tsx` 中的数据管理职责。
- 降低组件 props 传递复杂度。
- 优化会话、角色、模型、UI 状态同步。

### 3.2 表情启用控制

目标：允许角色只暴露部分表情给 AI。

可能改动：

- 在 Avatar 或 Character 数据中记录启用表情列表。
- 后端提供更新启用表情列表的接口。
- 前端在角色资产页提供表情启用开关。
- PromptService 只向 VRM 模式注入启用的表情。

当前状态：所有表情默认全部启用并传递给 AI。

### 3.3 多模态支持

- 评估图片、音频等多模态输入的产品形态。
- 明确模型能力识别、上传资源生命周期和对话上下文策略。
- 避免在基础聊天链路未稳定前引入过宽输入面。

## 4. 暂缓计划

以下方向暂不作为近期开发主线：

- 完整 VRM Director / 队列系统。
- 复杂场景对象控制。
- 桌宠模式。
- 资源 manifest / materialization。
- 会话级 system prompt snapshot。
- 长会话 FTS、搜索和压缩。
- 持久化 Agent 运行事件。
- 插件市场或通用外部 Memory Provider。

这些内容若后续重新进入主线，应先从 [未来改造参考](./未来改造参考.md) 中重新提案，再写入本路线图。

## 5. 不再推进

以下旧方案不再作为设计依据：

- 旧架构升级方案目录中的一次性重写计划。
- 旧 XML / 文本内嵌标签式 AI 指令双通道。
- 旧 `vrm_segment / vrm_complete` 输出链。
- 旧 `/messages` SSE 主链。
- 以“运行时升级方案”为独立蓝图的文档线。
