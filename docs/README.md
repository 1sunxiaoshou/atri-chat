# ATRI Chat 文档中心

本文档中心用于统一管理项目文档。文档按照使用场景分为「入门」「架构」「规划」三类，命名以中文主题名为主，避免中英混排和重复表述。

阅读时以 [系统设计](./02-架构/系统设计.md) 和各模块架构文档作为当前实现依据；规划、审计与复盘用于解释后续方向和历史决策，不能替代当前代码事实。

[路线图](./03-规划/路线图.md) 是唯一活跃计划来源。根 README、架构审计、未来参考和历史复盘不得再维护并行功能清单；候选事项只有写入路线图并给出验收标准后，才视为已经决定推进。

## 命名约定

- 目录使用 `数字-主题` 的形式，便于排序和扩展。
- 文档名称统一使用中文主题名，必要时保留通用缩写，如 `TTS`、`AI`、`VRM`。
- 模块内的局部说明优先保留在源码目录附近，例如 `frontend/components/**/README.md`。

## 维护原则

- 新增文档前，先判断是否属于现有目录，避免重复主题。
- 更新文档时，优先修改现有说明，不新建语义相近的副本。
- 根目录 [README.md](../README.md) 只保留导航和项目概览，详细内容统一收敛到 `docs/`。
- 当前事实写入 `02-架构/`，已决定事项写入路线图，候选建议与历史复盘保留在 `03-规划/` 并标明状态。
- 路线图条目必须包含目标和验收标准；已经完成的条目从路线图移除，并把稳定事实更新到对应架构文档。
- 带日期的审计数据是当时快照；引用文件、数量或测试结果前，应重新对照当前分支验证。
- 旧方案不能直接作为实现依据。
