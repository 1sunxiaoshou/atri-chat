## 已知问题与解决

### 问题 1: 导入路径错误
**错误**: `Failed to resolve import "../../ProviderIcon"`

**原因**: 文件在 `frontend/src/components/` 下，需要正确的相对路径

**解决**: 使用 `@/src/components/ProviderIcon` 别名路径

### 问题 2: 导出名称不匹配
**错误**: `does not provide an export named 'AWS'`

**原因**: `@lobehub/icons` 使用 `Aws` 而不是 `AWS`

**解决**: 
- 修正导入：`Aws`, `Bedrock`（不是 `AWS`）
- 移除不存在的导出：`Tongyi`, `IconProps`
- 使用 `React.ComponentType<any>` 替代 `IconProps`

## 优势总结

1. **统一视觉风格** - 所有图标来自同一设计系统
2. **自动深色模式** - 内置深浅色主题支持
3. **零存储开销** - 不需要上传和存储图片文件
4. **更快加载** - SVG 比图片更轻量
5. **易于维护** - 新增供应商只需添加映射关系
6. **代码简化** - 移除上传、存储、URL 构建等逻辑
7. **自动回退** - 未匹配的供应商显示首字母缩写

## 扩展指南

### 添加新供应商图标

1. 检查 `@lobehub/icons` 是否支持
2. 在 `PROVIDER_ICON_MAP` 中添加映射：

```typescript
import { NewProvider } from '@lobehub/icons';

const PROVIDER_ICON_MAP = {
  // ... 现有映射
  newprovider: NewProvider,
  alias: NewProvider,  // 支持别名
};
```

3. 如果不支持，组件会自动显示首字母回退
