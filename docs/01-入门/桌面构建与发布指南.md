# 桌面构建与发布指南

本文档说明 Windows 桌面端构建、打包、版本同步与发布产物约定。

## 常用命令

```bash
# 发布前检查
uv run python scripts/release.py check

# 交互式发布流程
uv run python scripts/release.py

# 完整发布构建
uv run python scripts/release.py release --format all --app-version <version>

# 仅同步版本
uv run python scripts/bump_version.py <version>

# 检查版本一致性
uv run python scripts/bump_version.py --check

# Tauri 构建
npm run tauri:build
```

## 发布产物

发布脚本产物放在 `release_package/`。

| 产物 | 用途 |
|---|---|
| `atri-chat-<version>-windows-x64-installer.exe` | Windows 安装包 |
| `atri-chat-<version>-windows-x64-portable.zip` | 便携版压缩包 |

当前支持格式：

- `installer`
- `portable`
- `all`

## 版本来源

版本号需要在以下文件中保持一致：

- `pyproject.toml`
- `package.json`
- `frontend/src-tauri/tauri.conf.json`
- `frontend/src-tauri/Cargo.toml`

推荐只通过脚本改版本：

```bash
uv run python scripts/bump_version.py 1.0.2
uv run python scripts/bump_version.py --check
```

## 运行模式

| 模式 | 后端 | 前端 | 数据目录 |
|---|---|---|---|
| 开发模式 | `uv run main.py` | Vite dev server | 仓库 `data/` |
| 安装版 | PyInstaller sidecar | Tauri WebView | 系统应用数据目录 |
| 便携版 | PyInstaller sidecar | Tauri WebView | 程序目录内数据 |

## 运行时契约

桌面模式下，Tauri 是运行时契约来源。Tauri 向 Python 后端注入：

- `ATRI_BACKEND_PORT`
- `ATRI_RUNTIME_MODE`
- `ATRI_APP_ROOT`
- `ATRI_DATA_ROOT`

Python 侧约定：

- `app_env` 从 `runtime_mode` 派生。
- `logs_dir` 固定为 `data_root/logs`。
- Python 创建日志、数据库、静态资产、模型、记忆和模态资源子目录。

不要重新引入跨进程的 `ATRI_APP_ENV` 或 `ATRI_LOGS_ROOT`，除非明确重设计启动契约。

## 组件职责

| 组件 | 职责 |
|---|---|
| Tauri | 启动窗口、解析运行时目录、分配后端端口、启动 sidecar |
| Python 后端 | 创建 FastAPI app、初始化数据库、挂载静态资源、注册 API |
| 发布脚本 | 同步版本、构建前检查、打包后端 sidecar、调用 Tauri 构建、整理产物 |

## 发布前检查

推荐发布前至少执行：

```bash
uv run python scripts/release.py check
npm --prefix frontend run type-check
npm --prefix frontend run lint
npm --prefix frontend run check-i18n
```

涉及后端逻辑时增加：

```bash
uv run ruff check .
uv run black --check .
uv run pytest
```
