# 国际化脚本指南

本文档说明前端翻译键检查与清理脚本的使用方式。

## 命令

从仓库根目录执行：

```bash
npm run check-i18n
npm run clean-i18n
```

等价的前端目录命令：

```bash
npm --prefix frontend run check-i18n
npm --prefix frontend run clean-i18n
```

## 脚本位置

- 脚本：`frontend/scripts/check-i18n.ts`
- 翻译文件：`frontend/locales/zh.json`、`frontend/locales/en.json`
- 扫描范围：`frontend/src`、`frontend/components`、`frontend/pages`、`frontend/hooks`、`frontend/store`、`frontend/contexts`

## 检查内容

`check-i18n` 会检查：

- `zh.json` 和 `en.json` 的键是否一致。
- 代码中使用的翻译键是否存在。
- 翻译文件中是否存在未使用键。

未使用键只作为警告，不会阻止命令成功。

## 清理规则

`clean-i18n` 会删除脚本判断为未使用的键。使用前建议确认：

- 键不是动态拼接出来的。
- 键不是由后端、配置或第三方组件间接使用。
- 当前工作区没有未保存的重要翻译修改。

删除后运行：

```bash
npm run check-i18n
npm --prefix frontend run type-check
```

## 翻译键约定

键名使用点分层：

```text
common.save
chat.send
settings.model.temperature
character.voice.select
```

常用前缀：

| 前缀 | 用途 |
|---|---|
| `common` | 通用按钮、状态、提示 |
| `chat` | 聊天界面 |
| `settings` | 设置页 |
| `character` | 角色管理 |
| `admin` | 管理页 |
| `vrm` | VRM 渲染与动作 |
| `provider` | 模型供应商 |
| `tts` | TTS |
| `asr` | ASR |

## 维护原则

- 新增 UI 文案时同步更新中英文翻译。
- 不使用 `t('key') || '默认值'` 作为兜底。
- 删除组件或页面后运行 `check-i18n`，必要时再运行 `clean-i18n`。
- 动态键应集中封装，避免脚本误判。
