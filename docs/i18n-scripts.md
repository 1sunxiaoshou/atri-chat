# 国际化脚本使用指南

## 概述

项目提供了两个 npm 脚本来管理国际化翻译键：

- `npm run check-i18n` - 检查翻译键的使用情况
- `npm run clean-i18n` - 删除未使用的翻译键

## 检查翻译键

```bash
npm run check-i18n
```

### 功能

1. **检查未定义的翻译键** - 代码中使用但翻译文件中不存在的键
2. **检查未使用的翻译键** - 翻译文件中定义但代码中未使用的键
3. **检查中英文一致性** - 确保 zh.json 和 en.json 的键完全一致
4. **检查错误用法** - 检测 `t('key') || '默认值'` 这种错误模式
5. **检查硬编码中文** - 发现代码中的硬编码中文字符串

## 清理未使用的翻译键

```bash
npm run clean-i18n
```

### 功能

自动删除 `zh.json` 和 `en.json` 中所有未使用的翻译键。

### 使用场景

- 重构代码后清理废弃的翻译键
- 定期维护翻译文件，保持代码整洁
- 删除实验性功能的翻译键

### 注意事项

⚠️ **使用前请确保：**

1. 代码已提交到 Git，以便在需要时回滚
2. 确认列出的"未使用"键确实不再需要
3. 某些键可能在动态场景中使用（如配置文件），脚本无法检测到

## 工作原理

### 扫描范围

脚本会扫描以下目录中的所有 `.ts`, `.tsx`, `.js`, `.jsx` 文件：

- `frontend/components/` - 所有组件
- `frontend/pages/` - 所有页面
- `frontend/*.tsx` - 根目录文件（如 App.tsx）

### 检测规则

1. **翻译键提取**
   - 匹配所有 `t('key')` 或 `t("key")` 调用
   - 支持模板字符串中的调用：`` `${t('key')} ${name}` ``
   - 支持带参数的调用：`t('key', { name: 'value' })`

2. **硬编码中文检测**
   - 检测字符串中的中文字符
   - 排除注释行
   - 排除 placeholder 等特殊场景

3. **错误用法检测**
   - 检测 `t('key') || '默认值'` 模式
   - 这种用法会导致翻译键检测失败

## 最佳实践

### 开发流程

1. **添加新功能前**
   ```bash
   npm run check-i18n
   ```
   确保当前翻译状态良好

2. **开发过程中**
   - 先在 `locales/*.json` 中添加翻译键
   - 然后在代码中使用 `t('key')`
   - 不要使用 `t('key') || '默认值'`

3. **提交代码前**
   ```bash
   npm run check-i18n
   ```
   确保没有未定义的翻译键

4. **定期维护**
   ```bash
   npm run clean-i18n
   ```
   清理未使用的翻译键

### 翻译键命名规范

- 使用小驼峰命名：`newChat`、`selectModel`
- 按功能模块分组：`sidebar.*`、`admin.*`、`chat.*`
- 保持键名简洁且语义清晰

### 常见前缀

- `sidebar.*` - 侧边栏相关
- `admin.*` - 管理后台相关
- `chat.*` - 聊天界面相关
- `settings.*` - 设置相关
- `app.*` - 应用全局相关

## 故障排除

### 脚本报告"未使用"但实际在使用

可能原因：
1. 使用了动态键：`t(dynamicKey)` - 脚本无法静态分析
2. 键在配置文件或数据库中使用
3. 键在注释或文档中

解决方案：
- 避免使用动态键
- 如果必须使用动态键，在代码中添加注释说明
- 手动保留这些键，不要运行 `clean-i18n`

### 脚本未检测到某些文件

检查：
1. 文件扩展名是否为 `.ts`, `.tsx`, `.js`, `.jsx`
2. 文件是否在 `components/`, `pages/` 或根目录
3. 文件是否在排除目录中（`node_modules`, `dist`, `.git`）

### 删除后发现需要恢复

使用 Git 恢复：
```bash
git checkout -- frontend/locales/zh.json frontend/locales/en.json
```

## 技术细节

### 正则表达式

```typescript
// 匹配 t() 调用（包括模板字符串中）
const tCallRegex = /\bt\s*\(\s*['"]([^'"]+)['"]\s*[,)}\]]/g;
```

### 删除算法

递归遍历翻译对象，删除指定的键，同时清理空的父对象。

### 文件格式

保持 JSON 文件的格式化（4 空格缩进）和末尾换行符。
